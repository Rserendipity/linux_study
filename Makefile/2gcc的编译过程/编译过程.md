源文件 -> 可执行文件的编译:
    1. gcc -E 
        进行头文件的包含，宏替换，宏展开
    2. gcc -S
        进行语义、语法等分析
        语法错误会在这个阶段被检测出来
        生成汇编代码
    3. gcc -c
        把汇编代码翻译成机器码(二进制)
        生成可连接的.o(目标)文件
    4. gcc 
        把生成的可连接文件连接起来
        链接动态/静态库生成最终的可执行文件

为什么Makefile把编译拆开：
    只有第一次编译才需要编译全部的文件
    项目修改时只需要编译有改动的文件，最后链接即可
    可以大大加快文件的编译速度
